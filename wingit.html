WingIt

Android App

Oct 2016 - Present

Version 2.0 on Google Play Store.

1 Summary

WingIt is an Android app that lets users experience spontaneous adventures anywhere, anytime. It creates a trip itinerary on the go, instead of planning the entire trip beforehand.

2 Design

Our main objective was to create a seamless experience for the user that simulated a real-time adventure. Therefore we came up with the following goals:

  1. Minimal screens and user interaction - the app should do all the hard work
  2. Clean screen designs to not distract the user from the adventure
  3. Screen transitions should flow on top of one another to create a cohesive and seamless experience
  4. The app should be an assistant or right-hand to the user - the real world is the actual adventure
  5. Follow Material Design guidelines

After countless iterations of wireframes and mockups, we came up with our current design, which was released as Version 2.0 on the Google Play Store. Even though it is not perfect, the designs achieved a large portion of our initial goals.

3 Technology + Methods

Android Studio, Google Places API, Firebase

When the app is launched, the user will see a "Start Your Adventure" screen with 2 options: Activity and Food. Once the user chooses a category, the "adventure" has begun.

The next screen shows the user 3 randomly generated places based on the user's current location. This was achieved by using Google Places API. The app will show more details and how far away each place is on a map, provided by Google Maps.

Once the user chooses a place to go, the app will redirect to Google Maps to navigate the user to the desired location. Once the user returns to the app, a map will show all the places the user has visited so far. The user can either continue or end the adventure from here.

The user may also see a timeline of his/her current adventure by tapping the carrot at the bottom of the screen.

Users may also visit previous adventures and see their timelines. We use a Firebase database for authentication and to store adventures.

4 Looking Forward

We are working on Version 3.0, where we hope to develop a cross-platform version of the app, and revamp the designs so that the users do not feel constrained or limited by the app. Some features to look forward to:

  1. More flexibility when generating places: filter by transportation method, number of choices, categories, food preferences, etc.
  2. Easier navigation to a location
  3. Recommendations along your adventure
